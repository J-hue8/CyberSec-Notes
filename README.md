## 第一天学习
- [2026-05-21] 1 漏洞点（网站的category参数直接拼接接到SQL查询语句中，没有做安全过滤）故完成第一个 SQL 注入实验：通过 `' OR 1=1--` 绕过分类过滤，理解了参数未过滤导致的 SQL 注入原理。
- [2026-05-21] 复现并深入理解 SQL 注入：通过构造 `' OR 1=1--`，成功检索隐藏数据。理解了攻击本质是改写后端 SQL 查询逻辑。修复建议：对用户输入做参数化查询。
- “**--**”是SQL里的**注释符**。
- [2026-05-21] 完成两个 SQL 注入实验：
  1. 利用 `' OR 1=1--` 绕过商品分类过滤，检索到所有隐藏商品。理解了单引号闭合和 `OR` 逻辑注入的原理。
  2. 利用 `administrator'--` 绕过管理员登录。理解了注释符 `--` 如何让密码验证部分失效。
  **核心收获**：SQL 注入的本质是通过构造恶意输入，改写后端 SQL 查询逻辑。这两次攻击分别利用了“不检查输入”和“拼接查询”的漏洞。正确的修复方法是使用参数化查询，将数据和代码彻底分离，而不是试图黑名单过滤。
